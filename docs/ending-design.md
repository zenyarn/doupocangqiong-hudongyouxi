# 游戏结局设计说明

## 一、结局类型概览

| 结局 | 触发条件 | 设计意图 |
|------|----------|----------|
| 💀 陨落 | 血量归零 | 负面结局，生存压力 |
| 🌱 凡人之路 | 修为 < 30 | 低成就，未真正突破 |
| ⚔️ 强者初成 | 修为 30~149 | 初步成功，证明自己 |
| 👑 王者之风 | 修为 150~399 | 中等成就，一方强者 |
| 🏆 宗师之名 | 修为 400~899 | 较高成就，开宗立派 |
| 🌟 尊者传说 | 修为 900~1799 | 高成就，大陆顶尖 |
| ✨ 圣者之路 | 修为 1800~3499 | 顶级成就，守护者 |
| 🔥 炎帝传说 | 修为 ≥ 3500 | 最高成就，斗帝 |

## 二、设计原则

### 1. 动态结局 vs 固定结局

**当前设计：动态结局**
- 结局由最终修为值决定
- 不与特定剧情节点绑定
- 玩家可在任何节点结束游戏

**为什么选择动态结局？**
```
问题：固定结局节点
├── 剧情描述暗示特定境界
├── 与玩家实际段位不符
└── 打破沉浸感

解决：动态判定
├── 剧情只描述事件，不描述境界
├── 根据最终修为动态判定
└── 保证结局与实力一致
```

### 2. 八种结局的层次设计

```
负面结局 (1种)
└── 陨落：生存压力，提醒玩家注意血量

正面结局 (7种) - 按修为梯度
├── 凡人之路：未能突破，但有努力
├── 强者初成：证明自己，非废物
├── 王者之风：称霸一方
├── 宗师之名：开宗立派
├── 尊者传说：大陆顶尖
├── 圣者之路：守护大陆
└── 炎帝传说：最高成就
```

### 代码实现
```typescript
export function getEndingByCultivation(cultivation: number): EndingInfo {
  if (cultivation >= 3500) return ENDING_CONFIG['炎帝传说'];
  if (cultivation >= 2500) return ENDING_CONFIG['圣者之路'];
  if (cultivation >= 900)  return ENDING_CONFIG['尊者传说'];
  if (cultivation >= 400)  return ENDING_CONFIG['宗师之名'];
  if (cultivation >= 150)  return ENDING_CONFIG['王者之风'];
  if (cultivation >= 30)   return ENDING_CONFIG['强者初成'];
  return ENDING_CONFIG['凡人之路'];
}
```

## 四、设计考量

### 优势
1. **灵活性**：玩家可在任何节点结束，都能获得合理结局
2. **一致性**：结局与实际实力匹配，避免"低段位高结局"bug
3. **可扩展**：新增剧情节点不影响结局逻辑
4. **策略性**：玩家需要平衡血量和修为积累

### 潜在问题及解决方案
| 问题 | 解决方案 |
|------|----------|
| 结局与剧情可能不连贯 | 结局描述使用通用语言，适配多种情况 |
| 玩家可能提前预测结局 | 保持选择的不确定性，随机事件影响数值 |
| 修为门槛难以达到 | 已降低斗帝门槛至3500，通过约20次斗气转化可达 |

## 五、数值门槛设计理由

### 为什么斗帝门槛是3500？

```
计算：
- 斗气转化：每次+15修为
- 理论最少转化次数：3500 ÷ 15 ≈ 233次
- 但剧情选项也会提供修为奖励

实际游戏：
- 约40-60个剧情节点
- 每个节点平均获得50-80修为
- 总计可获得2000-4800修为
- 加上斗气转化，可达3500+

调整：
- 原门槛8000过高，难以达成
- 降至3500后，合理路径可达
```

### 为什么血量归零=死亡？

1. **简化设计**：移除人脉信任，专注核心体验
2. **明确风险**：玩家清楚知道死亡条件
3. **策略维度**：战斗选项需权衡收益与风险
4. **原著还原**：修炼之路凶险，一着不慎满盘皆输

## 六、后续优化方向

1. **增加中间结局**：当前只有血量归零触发负面结局，可考虑增加其他条件结局
2. **结局与选择关联**：让关键选择影响结局类型，而不仅由修为决定
3. **多周目继承**：允许保留部分数值重新开始
4. **成就系统**：记录玩家达成的各种结局
