# 修炼系统重构设计方案 v2

## 一、核心问题分析

### 当前存在的问题
1. **数值过高**：斗帝需要8000修为，游戏流程难以达到
2. **机制混乱**：斗气、修为、人脉之间的关系不清晰
3. **结局触发错误**：故事节点固定跳转结局，与修为值脱钩
4. **属性冗余**：人脉信任的定位模糊，作用不明确

## 二、新设计方案

### 1. 属性系统重构

#### 新的三个属性

| 属性 | 范围 | 说明 |
|------|------|------|
| **血量** | 0-100 | 角色生命值，归零即死亡 |
| **斗气** | 0-100 | 修炼资源，蓄满自动转化为修为 |
| **丹药** | 0-100 | 消耗品，用于恢复血量或斗气 |

#### 隐藏属性

| 属性 | 说明 |
|------|------|
| **修为** | 累计值，决定当前段位 |

### 2. 转化机制

**核心机制：斗气蓄满 → 自动转化修为**

```
当斗气达到100时：
1. 斗气清零
2. 修为 +15
3. 显示转化提示："斗气圆满，修为精进！"
```

**丹药使用机制：**
- 使用丹药恢复血量：+20血量，-10丹药
- 使用丹药恢复斗气：+30斗气，-15丹药

### 3. 段位与修为对应（降低门槛）

| 段位 | 所需修为 | 说明 |
|------|---------|------|
| 斗之气 | 0 | 初始境界 |
| 斗者 | 30 | 初入修炼 |
| 斗师 | 80 | 小有成就 |
| 大斗师 | 150 | 略有名气 |
| 斗灵 | 250 | 一方强者 |
| 斗王 | 400 | 称霸一方 |
| 斗皇 | 600 | 威震一方 |
| 斗宗 | 900 | 宗师级别 |
| 斗尊 | 1300 | 尊者之境 |
| 半圣 | 1800 | 半步圣人 |
| 斗圣 | 2500 | 圣者之境 |
| 斗帝 | 3500 | 巅峰之境 |

**设计思路**：
- 每次斗气满转化约+15修为
- 从斗之气到斗帝需要约233次转化
- 考虑到选择也会直接增加修为，实际游戏流程约50-100个节点可达到

### 4. 结局系统重构

#### 唯一负面结局
- **陨落结局**：血量归零时触发

#### 正面结局（基于最终修为）

| 最终段位 | 结局类型 | 结局描述 |
|---------|---------|---------|
| 斗帝 | 炎帝传说 | 传说中的炎帝，振兴种族 |
| 斗圣 | 圣者之路 | 达到圣者境界，守护大陆 |
| 半圣~斗尊 | 尊者传说 | 成为尊者级别的强者 |
| 斗宗~斗皇 | 宗师之名 | 一代宗师，名震一方 |
| 斗王~斗灵 | 王者之风 | 成为王者，称霸一方 |
| 大斗师~斗师 | 强者初成 | 初成强者，有所成就 |
| 斗者~斗之气 | 凡人之路 | 修炼未成，仍是凡人 |

### 5. 游戏流程设计

#### 节点结束判定
- 当故事节点走到最后一个选项时（没有后续节点）
- 检查血量是否归零 → 陨落结局
- 否则根据当前修为/段位显示对应结局

#### 移除固定结局节点
- 删除 `ending_emperor`, `ending_hero`, `ending_dark` 等固定节点
- 结局由修为值动态决定

### 6. 选择效果调整

每次选择的影响：
- **血量变化**：战斗失败、受伤
- **斗气变化**：修炼、战斗胜利
- **丹药变化**：获得/消耗
- **修为变化**：重大突破、特殊事件

## 三、实现细节

### 状态管理
```typescript
interface GameState {
  // 显示属性
  hp: number;           // 血量 0-100
  qi: number;           // 斗气 0-100
  medicine: number;     // 丹药 0-100
  
  // 隐藏属性
  cultivation: number;  // 修为（累计）
  
  // 段位（由修为自动计算）
  level: DouQiLevel;
  
  // 游戏状态
  currentNodeId: string;
  visitedNodes: string[];
  choices: Record<string, string>;
}
```

### 自动转化逻辑
```typescript
function checkQiConversion(state: GameState): GameState {
  if (state.qi >= 100) {
    return {
      ...state,
      qi: 0,
      cultivation: state.cultivation + 15,
      showConversionMessage: true,
    };
  }
  return state;
}
```

### 结局判定逻辑
```typescript
function determineEnding(cultivation: number): EndingType {
  if (cultivation >= 3500) return '炎帝传说';
  if (cultivation >= 2500) return '圣者之路';
  if (cultivation >= 1800) return '尊者传说';
  if (cultivation >= 900) return '宗师之名';
  if (cultivation >= 400) return '王者之风';
  if (cultivation >= 150) return '强者初成';
  return '凡人之路';
}
```
