# 斗破苍穹：炎帝之路 - 修炼系统设计文档

## 一、核心设计理念

### 1. 晋升机制
采用**经验值累计系统**，斗气值作为经验值，累计到一定数值自动晋升段位。

### 2. 段位体系（简化版）
| 段位 | 所需累计斗气值 | 晋升条件 |
|------|---------------|---------|
| 斗之气 | 0 (初始) | 累计斗气值达到50 |
| 斗者 | 50 | 累计斗气值达到150 |
| 斗师 | 150 | 累计斗气值达到300 |
| 大斗师 | 300 | 累计斗气值达到500 |
| 斗灵 | 500 | 累计斗气值达到800 |
| 斗王 | 800 | 累计斗气值达到1200 |
| 斗皇 | 1200 | 累计斗气值达到1800 |
| 斗宗 | 1800 | 累计斗气值达到2500 |
| 斗尊 | 2500 | 累计斗气值达到3500 |
| 半圣 | 3500 | 累计斗气值达到5000 |
| 斗圣 | 5000 | 累计斗气值达到8000 |
| 斗帝 | 8000 | 最终境界 |

### 3. 状态值设计

#### 斗气值（经验值）
- **作用**：决定段位晋升的核心指标
- **获取方式**：修炼、丹药、战斗胜利
- **消耗方式**：战斗失败、使用技能
- **范围**：0-100（当前值）+ 累计值（决定段位）

#### 丹药储备
- **作用**：可用来恢复斗气值或交易
- **范围**：0-100

#### 人脉信任
- **作用**：影响NPC帮助概率、触发隐藏剧情
- **范围**：0-100
- **特殊**：过低可能触发黑化结局

### 4. UI显示设计

#### 顶部状态栏
- **段位**：只显示当前段位名称（如"斗者"、"斗王"），不显示星级
- **进度条**：显示到下一阶段所需的累计斗气值进度
- **三项数值**：当前斗气值 / 丹药储备 / 人脉信任

#### 场景背景
根据不同场景使用不同的古风配色：
- 乌坦城：土黄色、古铜色
- 魔兽山脉：墨绿色、深棕色
- 云岚宗：青白色、淡蓝色
- 沙漠：沙黄色、暗红色
- 洞穴：深灰色、暗紫色
- 修炼：紫金色、玄青色

## 二、剧情与数值的对应关系

### 1. 设计原则
- **剧情独立于数值**：剧情描述不直接提及"突破到XX境界"
- **数值驱动晋升**：段位晋升完全由累计斗气值决定
- **剧情暗示而非明示**：用"感受到力量增长"等模糊描述

### 2. 节点设计调整
移除以下类型的节点：
- 直接描述"突破到XX境界"的节点
- 与数值系统冲突的硬性晋升节点

保留以下类型的节点：
- 场景描述节点（乌坦城嘲讽、魔兽山脉探险等）
- 选择影响数值的节点
- 关键剧情节点（药老现身、三年之约等）

### 3. 数值调整说明
每次选择的斗气值变化需要重新平衡：
- 普通修炼：+5~15
- 战斗胜利：+10~20
- 获得丹药：+3~10
- 战斗失败：-10~20
- 使用丹药恢复：+15~30

## 三、结局判定

### 斗帝结局
- 累计斗气值达到8000
- 人脉信任 > 50

### 英雄结局
- 累计斗气值达到5000
- 人脉信任 > 70

### 黑化结局
- 人脉信任 < 10
- 在特定节点做出黑化选择

### 陨落结局
- 当前斗气值降为0
- 战斗失败且无丹药恢复

## 四、技术实现

### 1. 状态管理
```typescript
interface GameState {
  level: DouQiLevel;          // 当前段位
  totalDouQi: number;         // 累计斗气值（决定段位）
  currentDouQi: number;       // 当前斗气值（0-100）
  medicineReserve: number;    // 丹药储备
  trustValue: number;         // 人脉信任
  // ...
}
```

### 2. 晋升判定
每次斗气值变化后，检查累计值是否达到晋升门槛：
```typescript
function checkLevelUp(totalDouQi: number): DouQiLevel {
  if (totalDouQi >= 8000) return '斗帝';
  if (totalDouQi >= 5000) return '斗圣';
  // ...
}
```

### 3. 显示逻辑
- 进度条 = (累计斗气值 - 当前阶段起点) / (下一阶段起点 - 当前阶段起点)
- 段位名称直接显示，无星级
